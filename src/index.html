Therion: new approach to cave surveying

To existing cave-mapping software adds Therion following features:
  • all survey data are RELATIVE to survey stations
  • produces complete 2D plan and elevation maps of cave in PDF (and eventually SVG) format
  • simple changes of map layout via loading of different MetaPost symbol packages (PDF) or CSS (SVG)
  • 2D maps are adapted for particular output scale (level of abstraction, non-linear scaling of symbols and texts)
  • 3D model is generated from 2D plan, elevation and cross-sections data
  • supports national encodings and UNICODE
  • input is a human readable text file, that can be easily converted to any other format
  • zipped sources of 1 km cave passages take only about 25 kB of your storage
  • limited WYSIWYG support; but you get what you want ;-)
  • it's FREE and OPENSOURCE, like all used programs and libraries

Brief description:
  • Therion—Data processing engine. (Requires Survex for loop closure; Perl interpreter for running thPdf; MetaPost and pdfTeX 1.0 for 2D map export; Tcl/Tk with BWidget for xtherion)

    [source 03/01/05] [linux executable 02/11/06] [win32 executable 02/11/06] [documentation 02/12/03]

  • xtherion—Tcl/Tk user interface. It consists of text editor (to enter survey data with a minimal typing), map editor and compiler. Features of text editor: easy entering of survey data in specified order, table formatting and Unicode support. Features of map editor: fully editable bezier curves, displaying of scanned survey sketches at the background. Features of compiler: F9 in xtherion will run therion to make current project. You may use arbitrary plain text editor instead of it. (All data files, including 2D maps, are in hand-editable text format.)

2D output

All map data are relative to survey stations. This implies, that your drawings are allways up-to-date. You don't need to re-edit the whole map after any dramatic loop closure. ;-)

Special bonus

You may improve your survey drawings by using our Therion Protractor in the cave. Therion distorts the map much less in the process of adjusting of the 2D data (entered in xtherion with scanned sketches in the background) to survey stations after loop closure, when data are drawn in scale already in the cave. (Not to say, that you have ready-to-use map when you leave the cave, as well...)

General informations

Software requirements: Survex, MetaPost, pdfTeX and standard unix equipement, like Perl interpreter, C and C++ compiler, Tcl/Tk with BWidget package.

Supported platforms: Linux/Unix, MacOS X, Windows
